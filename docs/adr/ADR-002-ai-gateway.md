# ADR-002: Внутренний AI Gateway как единственный LLM-провайдер

**Status:** Accepted | **Date:** 2026-04-23 | **Version:** 2.0.0 | **Author:** AI Engineer, InfoSec Specialist  
**Supersedes:** ADR-002 v1.0.0 (OpenRouter как прокси к LLM)

---

## Контекст

Сервис требует вызовов LLM для генерации текста протокола КИ и проверки консистентности. Протоколы клинических исследований содержат конфиденциальную медицинскую и коммерческую информацию (наименования препаратов в разработке, данные пациентов, стратегические эндпоинты). Передача таких данных внешним LLM-сервисам недопустима по политике информационной безопасности.

## Рассмотренные варианты

| Вариант | За | Против |
|---|---|---|
| **Внутренний AI Gateway** (выбрано) | Данные не покидают контур; OpenAI-compatible API; InHouse/Qwen3.5-122B | Зависимость от доступности внутренней инфраструктуры |
| **OpenRouter / внешние LLM** | Широкий выбор моделей | Данные КИ передаются третьим сторонам — **запрещено политикой ИБ** |
| **OpenAI / Anthropic напрямую** | Высокое качество моделей | Данные КИ передаются третьим сторонам — **запрещено политикой ИБ** |
| **Локальная модель без Gateway** | Полная изоляция | Сложность деплоя; отсутствие стандартизированного API |

## Решение

Используется **исключительно внутренний AI Gateway** с моделью `InHouse/Qwen3.5-122B`.  
Любые вызовы к внешним LLM запрещены в production-коде.

## Обоснование (Rationale)

1. **Безопасность данных:** протоколы КИ содержат конфиденциальную информацию — передача во внешние сервисы нарушает политику ИБ и потенциально регуляторные требования (персональные данные, коммерческая тайна)
2. **Совместимость:** AI Gateway предоставляет OpenAI-compatible API (`POST /v1/chat/completions`) — интеграция через `httpx.AsyncClient` без изменения структуры кода
3. **Контроль:** модель и параметры настраиваются централизованно на уровне Gateway
4. **Соответствие NFR-08:** требование использовать только локальные модели зафиксировано в функциональных требованиях

## Последствия

- `AI_GATEWAY_URL` — обязательная переменная окружения
- `AI_GATEWAY_API_KEY` — обязательная переменная окружения
- `AI_GATEWAY_MODEL` — модель по умолчанию: `InHouse/Qwen3.5-122B`
- Все AI-вызовы логируются в `audit_log`: timestamp, model, section_type, duration_ms
- Retry-логика: 3 попытки с exponential backoff через `tenacity` при сетевых ошибках
- При недоступности Gateway — возвращается HTTP 503 с корректным error body (не 500)
