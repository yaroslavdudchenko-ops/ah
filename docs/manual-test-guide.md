## Сценарии тестирования

### Сценарий A — Happy Path (Admin): Полный цикл

1. Войти как **admin** / `admin123`
2. Нажать «Новый протокол», заполнить **Протокол 1 (BCD-100)** по таблице выше
3. Нажать «Создать протокол» → ожидаем редирект на страницу протокола
4. Нажать «Генерировать протокол» → наблюдать анимацию Synthia Orb
5. После завершения: убедиться, что появились ≥7 разделов в сайдбаре
6. Нажать «Черновик» → открывается модал со всеми разделами
7. Нажать «Печать / PDF» → проверить наличие даты печати и watermark
8. Нажать «GCP-проверка» → получить score и список замечаний
9. Нажать «md» → файл скачивается
10. Нажать «docx» → файл скачивается
11. Открыть вкладку «Аудит» → видны события generate, create

**Ожидаемый результат:** Все шаги завершены без ошибок, ≥7 разделов, файлы скачиваются.

---

### Сценарий B — Employee: Ограничение удаления

1. Выйти, войти как **employee** / `employee123`
2. Открыть список протоколов
3. Убедиться: **кнопка корзины (удаления) ОТСУТСТВУЕТ** на всех протоколах
4. Создать новый протокол (Протокол 2 — BCD-089)
5. Нажать «Генерировать» → доступно
6. Убедиться: в шапке нет кнопки «Удалить»

**Ожидаемый результат:** Employee создаёт и просматривает, но не удаляет.

---

### Сценарий C — Auditor: Только чтение

1. Выйти, войти как **auditor** / `auditor123`
2. Открыть список → протоколы видны, кнопок «+» и удаления нет
3. Открыть протокол → разделы доступны для чтения
4. Попробовать нажать «Генерировать» → кнопка отсутствует
5. Открыть `/audit` → журнал аудита доступен
6. Нажать «Экспорт PDF» на странице аудита → файл скачивается

**Ожидаемый результат:** Полный read-only режим.

---

### Сценарий D — Поиск и фильтры

1. Войти как **admin**
2. На главной странице, в строке поиска ввести `BCD` → автокомплит показывает варианты
3. Выбрать протокол из автокомплита → список фильтруется
4. Нажать «Фильтры» → выбрать «Фаза III» → список сужается
5. Добавить фильтр «Статус: Черновик»
6. Убедиться: счётчик на кнопке «Фильтры» показывает «2»
7. Нажать «Сбросить всё» → все протоколы снова видны

**Ожидаемый результат:** Поиск и фильтры работают, комбинируются, сбрасываются.

---

### Сценарий E — Теги

1. Открыть существующий протокол
2. В блоке «Теги» нажать «Изменить»
3. Ввести тег `тест-тег-2026`, нажать Enter → чип появился
4. Нажать «Сохранить»
5. Обновить страницу (F5) → тег сохранён
6. Вернуться в список → тег виден рядом с протоколом
7. Кликнуть на тег → список фильтруется

**Ожидаемый результат:** Тег создаётся, сохраняется, фильтрует.

## Сценарий: Лок, архивирование, ревизия

| Шаг | Логин | Действие | Ожидание |
|---|---|---|---|
| L-01 | admin | Создать протокол BCD-XXX | Протокол в статусе «Черновик», кнопка «Генерировать» активна |

| L-05 | employee | Открыть тот же протокол | Видна кнопка «Создать ревизию» |
| L-06 | employee | Нажать «Создать ревизию» → подтвердить | Редирект на новый протокол; старый → status=archived, тег Archive |
| L-07 | admin | Открыть список протоколов → Фильтры → Статус «В архиве» | Показывается заархивированный протокол |
