## Пользователи и контекст

| Пользователь | Контекст | Главная задача |
|---|---|---|
| Исследователь / КМ | Рабочий стол, Chrome/Edge | Создать черновик протокола за минимальное время |
| Медицинский ревьюер | Читает, не пишет | Найти список открытых вопросов, сравнить версии |

**Ключевой инсайт:** пользователи — специалисты с медицинским образованием, работают с длинными документами. UI должен быть профессиональным, не перегруженным, с чёткой навигацией по разделам.

## User Flows

### Flow 1: Создание и генерация протокола

```
Главная → [+ Новый протокол]
  → Выбор шаблона (фаза + тип дизайна)
  → Форма параметров (многошаговая: 3 шага)
      Шаг 1: Основные данные (название, фаза, индикация, препарат/МНН)
      Шаг 2: Дизайн (популяция, конечные точки, длительность, дозирование)
      Шаг 3: Критерии исключения (динамический список)
  → Предпросмотр параметров + [Сгенерировать]
  → Экран генерации (прогресс секция за секцией)
  → Просмотр готового протокола
```

### Flow 2: Работа с версиями

```
Страница протокола → История версий (sidebar)
  → Открыть версию v0.1
  → [Сравнить с v0.2] → DiffViewer
  → [Создать новую версию] → перегенерация выбранных секций
```

### Flow 3: Проверка и экспорт

```
Страница протокола → [Проверить консистентность]
  → Панель с найденными противоречиями + GCP hints
  → [Исправить] → редактирование секции inline
  → [Экспортировать] → выбор формата (MD / HTML / DOCX)
  → Скачивание файла
```

## Дизайн-система

### Цветовая палитра

| Токен | Цвет | Использование |
|---|---|---|
| `primary` | `#2563EB` (blue-600) | Кнопки, активные элементы |
| `success` | `#16A34A` (green-600) | Статус "завершено", GCP OK |
| `warning` | `#D97706` (amber-600) | Открытые вопросы, warnings |
| `danger` | `#DC2626` (red-600) | Ошибки, противоречия high severity |
| `neutral` | `#6B7280` (gray-500) | Второстепенный текст |
| `surface` | `#F9FAFB` (gray-50) | Фон секций |

### Типография

| Элемент | Размер | Вес |
|---|---|---|
| H1 (название протокола) | 24px | 700 |
| H2 (раздел протокола) | 18px | 600 |
| H3 (подраздел) | 15px | 600 |
| Body | 14px | 400 |
| Caption (версия, дата) | 12px | 400 |

### Компоненты

| Компонент | Использование |
|---|---|
| `Badge` | Фаза (I/II/III), статус (draft/review/final) |
| `Spinner` | AI генерация в процессе |
| `Toast` | Успех/ошибка операций |
| `ProgressSteps` | Многошаговая форма |
| `DiffLine` | Добавленная/удалённая строка в diff |
| `SeverityBadge` | high/medium/low для consistency issues |

## UX-требования

| Требование | Описание |
|---|---|
| AI-генерация | Показывать прогресс секция за секцией, не просто спиннер |
| Длинные документы | Sticky sidebar с навигацией по секциям |
| Diff | Цветовая разметка: зелёный — добавлено, красный — удалено |
| Экспорт | Не открывать новую вкладку — скачивать файл напрямую |
| Ошибки AI | Показывать конкретное сообщение + кнопку повтора |
| Watermark | В экспортированных файлах: "FOR DEMONSTRATION ONLY" |
| Дисклеймер | На каждой странице: "AI-generated draft. Requires qualified person review." |

## Accessibility

- Контраст текста ≥ 4.5:1 (WCAG AA)
- Все интерактивные элементы доступны с клавиатуры
- Aria-labels для иконок без текста
- Focus ring виден на всех элементах
