# Онбординг в Dokploy

Инструкция для пользователей, которые впервые начинают работу с Dokploy.
Этот текст должен быть выведен пользователю на шаге 7 (Step 7) основного воркфлоу.

---

## Что такое Dokploy

Dokploy — это платформа для деплоя приложений через Docker Compose. Она берёт на себя маршрутизацию (Traefik), сертификаты, мониторинг и управление переменными окружения. Вам нужно только подготовить репозиторий — всё остальное настраивает администратор.

## Как получить доступ

### Шаг 1 — Запросите права

1. Откройте корпоративную систему управления доступом (IDM) и запросите роль **Dokploy - project member**.
2. В обосновании укажите ссылку на ваш репозиторий и кратко опишите, что будете деплоить.
3. Запрос пройдёт согласование и автоматически создаст задачу для платформенного инженера.

### Шаг 2 — Активируйте аккаунт

1. На корпоративную почту придёт письмо со ссылкой для регистрации.
2. Перейдите по ссылке, задайте пароль и войдите в Dokploy.
3. **Сообщите инженеру** (в рамках Jira-тикета), что аккаунт создан — права к конкретному проекту выдаются вручную.

> После того как инженер выдаст права к проекту, вы увидите его в своём дашборде.

## Что вы увидите в дашборде

### Репозиторий уже подключён

Подключение репозитория делает **администратор** через Git-провайдер с токеном. Вы увидите:
- **Owner** — группа/неймспейс (например, `sandbox`)
- **Repository** — название вашего репозитория
- **Branch** — ветка для деплоя (обычно `master` или `main`)

Предупреждение _"connected through a git provider that you don't have access to"_ — это **нормально**. Деплой работает, просто вы не можете менять настройки подключения.

> **НЕ нажимайте "Disconnect Repository"!** Это отключит репозиторий, и восстановить связь сможет только администратор.

### Вкладки приложения

| Вкладка | Назначение |
|---------|------------|
| **General** | Основные настройки, кнопки Deploy / Reload / Stop |
| **Environment** | Переменные окружения (секреты, пароли, URL базы данных) |
| **Domains** | Настройка доменов и URL |
| **Deployments** | История деплоев. Если что-то сломалось — смотрите здесь (кнопка View) |
| **Logs** | Логи работающего приложения |
| **Monitoring** | Графики CPU, памяти, диска, сети |

## Как настроить переменные окружения

1. Перейдите на вкладку **Environment**.
2. Добавьте переменные в формате `KEY=value`, по одной на строку.
3. Нажмите **Save**.
4. Перейдите на **General** → нажмите **Deploy** для применения.

Переменные записываются в `.env` файл рядом с `docker-compose.yml`. Ваш compose-файл ссылается на них через `${VAR_NAME}`.

**Какие переменные нужно задать** — смотрите файл `.env.example` в вашем репозитории.

## Как настроить домен

### Быстро, для тестирования (HTTP)

1. Вкладка **Domains** → **Add Domain**.
2. В **Service Name** выберите ваш сервис из списка.
3. Нажмите кнопку 🎲 рядом с **Host** — Dokploy сгенерирует адрес вида `your-app-xxxxxx.traefik.me`.
4. В **Container Port** укажите порт приложения (например, `3000`).
5. **HTTPS не включайте** — `traefik.me` работает только по HTTP.
6. Нажмите **Create/Update** → на вкладке **General** нажмите **Deploy**.

> `traefik.me` — публичный HTTP-адрес без SSL. Подходит только для временного тестирования внутри компании.

### С HTTPS (через DevOps)

1. Создайте заявку в DevOps на DNS-запись для нужного поддомена.
2. Когда DNS готов, перейдите в **Domains** → **Add Domain**.
3. В **Host** введите ваш домен.
4. В **Container Port** укажите порт приложения.
5. Включите **HTTPS**.
6. В **Certificate** выберите **none** (если настроен wildcard-сертификат) или Let's Encrypt.
7. Нажмите **Create/Update** → **Deploy**.

Источник: [Dokploy Domains](https://docs.dokploy.com/docs/core/docker-compose/domains)

## Как деплоить

### Ручной деплой
Вкладка **General** → кнопка **Deploy**.

### Автодеплой
Если администратор включил Auto Deploy и настроил webhook, деплой произойдёт автоматически при push в указанную ветку.

Проверить статус деплоя: вкладка **Deployments**.

### Что делать если деплой упал

1. Перейдите на **Deployments** → найдите последний деплой → **View**.
2. Прочитайте лог сборки — там будет ошибка.
3. Типичные причины:
   - Отсутствует обязательная переменная окружения → задайте на вкладке Environment
   - Ошибка сборки Docker-образа → проверьте Dockerfile и зависимости
   - Healthcheck не проходит → проверьте, что эндпоинт `/health` возвращает 200
4. После исправления нажмите **Deploy** снова.

## Чего делать НЕ нужно

| Действие | Почему нельзя |
|----------|--------------|
| Нажимать "Disconnect Repository" | Отключит репозиторий, восстановить может только админ |
| Менять настройки Git Provider | У вас нет доступа, и это может сломать деплой |
| Использовать `./` пути для томов в compose | Файлы пропадут при следующем деплое (AutoDeploy делает `git clone`). Используйте именованные volumes |
| Использовать `ports: "3000:3000"` | Привязка к хосту вызовет конфликт с другими приложениями |
| Задавать `container_name` | Сломает логи, метрики и мониторинг в Dokploy |
| Использовать `privileged: true` | Запрещено политикой безопасности платформы |
| Коммитить реальные секреты | Секреты задаются через Environment в Dokploy UI |
