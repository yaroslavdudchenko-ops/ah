# Prompt: Consistency & Contradiction Check

**Version:** 1.0.0 | **Date:** 2026-04-23

---

```
Ты — эксперт по качеству клинических протоколов. Проанализируй протокол на наличие логических противоречий и несоответствий терминологии.

ПРОТОКОЛ ДЛЯ АНАЛИЗА:
{protocol_content}

ПАРАМЕТРЫ ПРОТОКОЛА:
- МНН препарата: {inn}
- Торговое название: {drug_name}
- Фаза: {phase}
- Primary endpoint: {primary_endpoint}

ПРОВЕРЬ СЛЕДУЮЩИЕ ТИПЫ ПРОТИВОРЕЧИЙ:
1. Primary endpoint vs. план статистического анализа (метод анализа должен соответствовать типу endpoint)
2. Критерии включения/исключения vs. заявленная популяция (возраст, состояние, ECOG и т.д.)
3. Длительность исследования vs. график визитов
4. Режим дозирования vs. фаза исследования (фаза I — простые режимы)
5. Размер выборки vs. статистическая мощность

ПРОВЕРЬ КОНСИСТЕНТНОСТЬ ТЕРМИНОЛОГИИ:
- Название препарата (МНН и торговое) должно быть единообразным по всему документу
- Названия конечных точек должны совпадать во всех разделах

ФОРМАТ ОТВЕТА (строгий JSON, snake_case):
{
  "contradictions": [
    {
      "type": "endpoint_mismatch | population_inconsistency | duration_mismatch | dosing_inconsistency | sample_size | terminology",
      "severity": "high | medium | low",
      "sections": ["section1", "section2"],
      "description": "Описание противоречия на русском языке",
      "suggestion": "Конкретная рекомендация по исправлению"
    }
  ],
  "terminology_issues": [
    {
      "term": "найденный вариант написания",
      "preferred": "предпочтительная форма",
      "sections": ["section1"]
    }
  ],
  "compliance_score": 0-100,
  "summary": "Общий вывод о консистентности протокола"
}

Если противоречий нет — верни пустые массивы и compliance_score: 100.
```
